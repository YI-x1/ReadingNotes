# 文献分析协作规则

## 核心原则

- Zotero 是本工作区的唯一可信文献库。
- 不允许编造作者、年份、标题、期刊、会议、DOI、URL、页码、卷期号、citation key 或任何其他文献信息。
- 所有综述结论必须能够追溯到具体文献，并保留 citation key。
- 输出优先使用中文；如需保留英文术语，应同时保证中文解释清晰。
- 生成文献矩阵、综述、PPT 或 slides outline 时，必须保留 citation key。
- 不删除原始 PDF，不覆盖用户已有批注，不修改 Zotero 原始导出数据，除非用户明确要求。

## 证据与引用要求

- 只有在 Zotero 导出数据、PDF、标注或用户提供的材料中出现过的信息，才可作为文献事实使用。
- 如果作者、年份、DOI、页码或结论无法确认，应明确标注为“待核查”，不要自行补全。
- 直接引文必须保留页码或位置线索；如果没有页码，应说明来源位置不可确认。
- 综述中的观点应区分为：文献发现、作者主张、用户分析、Codex 综合推断。

## 写作要求

- 优先生成结构清晰、可继续编辑的中文内容。
- 文献综述应以研究问题为中心，而不是简单按论文逐篇罗列。
- 每个关键论点应尽量连接到 evidence matrix 或 paper card。
- 生成 PPT 大纲时，应保留每页对应的 citation key，便于后续核查。

## 文件安全要求

- 不删除 `data/pdfs/` 中的原始 PDF。
- 不删除 `data/zotero/` 中的 Zotero 导出文件。
- 不进行批量覆盖，除非用户明确确认。
- 自动化脚本应优先生成新文件或可审阅的草稿。
